# Eggs

Spawn one tiny animated desktop companion. It can idle, move, sleep, eat, play, roar, attack, and generally mind its own tiny business.

This is a standalone Codex skill that launches an animated 2D sprite companion onto your desktop when you invoke:

```text
/eggs
```

It uses the sprite sheet generated by this repo's sprite toolbox:

```text
eggs/assets/dino.png
```

The optional remote sprite library server lives outside the skill in `server/`. The skill can connect to a server, but installing the skill does not install or run a server.

## Companion Capabilities

- Roams around your desktop in a transparent always-on-top window.
- Can be dragged with the mouse to reposition it.
- Uses a native Swift/Cocoa overlay on macOS.
- Falls back to Python/Tk outside macOS, if your Python has Tkinter.
- Plays one sprite-sheet row per state: unborn, ready, hatching, hatched, walk, sleep, eat, drink, play, roar, and attack.
- Stores runtime state in `~/.codex/eggs/`.

## Asset Lookup

When someone installs this skill, the runtime does not read from this repo's original absolute path. It resolves assets relative to the installed skill directory:

- Custom/user-installed sprite: `~/.codex/eggs/<sprite>.png`
- Custom/user-installed metadata: `~/.codex/eggs/<sprite>.json`
- Bundled fallback sprite: `<installed-skill>/assets/<sprite>.png`
- Bundled fallback metadata: `<installed-skill>/assets/<sprite>.json`
- Procedural fallback: drawn by the script if no PNG is available

`eggs/assets/dino.json` is portable and is read by the animation runtime. Its `image` field is `dino.png`, relative to the JSON file's own directory, not `/Users/...`; its `frameWidth` and `frameHeight` define how the PNG is sliced and how large the desktop window should be. Keep generated metadata this way so the skill can be moved or installed on another machine.

The current sprite and animation state are stored in `~/.codex/eggs/state.json`:

```json
{
  "sprite": "dino",
  "state": "walk"
}
```

Custom animation row names are stored in `~/.codex/eggs/config.json`. `row` is zero-based and points to the spritesheet row; `loop` can be `true`, `false`, or a number of loops:

```json
{
  "animations": {
    "dino": {
      "idle": { "row": 3, "loop": true },
      "walk": { "row": 4, "loop": true },
      "attackOnce": { "row": 10, "loop": false },
      "roar3": { "row": 9, "loop": 3 }
    }
  }
}
```

## Companion Commands

Start the companion:

```bash
cd eggs
python3 scripts/egg_desktop.py start
```

Stop the companion:

```bash
cd eggs
python3 scripts/egg_desktop.py stop
```

Check status:

```bash
cd eggs
python3 scripts/egg_desktop.py status
```

Apply runtime code changes or restart it:

```bash
cd eggs
python3 scripts/egg_desktop.py restart
```

Switch animation state:

```bash
cd eggs
python3 scripts/egg_desktop.py state unborn
python3 scripts/egg_desktop.py state ready
python3 scripts/egg_desktop.py state hatching
python3 scripts/egg_desktop.py state hatched
python3 scripts/egg_desktop.py state walk
python3 scripts/egg_desktop.py state sleep
python3 scripts/egg_desktop.py state eat
python3 scripts/egg_desktop.py state drink
python3 scripts/egg_desktop.py state play
python3 scripts/egg_desktop.py state roar
python3 scripts/egg_desktop.py state attack
```

Switch sprite:

```bash
cd eggs
python3 scripts/egg_desktop.py sprite dino
python3 scripts/egg_desktop.py state walk dino
python3 scripts/egg_desktop.py state attackOnce dino
```

Install a replacement sprite sheet:

```bash
cd eggs
python3 scripts/egg_desktop.py spritesheet /path/to/dino.png dino
python3 scripts/egg_desktop.py restart
```

## Remote Interaction

Remote interaction is opt-in and off by default. It stores anonymous client identity in `~/.codex/eggs/client.json`, remote settings in `~/.codex/eggs/remote.json`, and cached downloaded remote peer indexes under `~/.codex/eggs/remote/<content_id>/` with shared blob files under `~/.codex/eggs/remote/blobs/`.

Configure a server:

```bash
cd eggs
python3 scripts/egg_desktop.py remote server http://localhost:8787
```

Upload the current local sprite assets:

```bash
cd eggs
python3 scripts/egg_desktop.py remote upload dino
```

The uploaded sprite name is also remembered in `~/.codex/eggs/remote.json`, and room interaction uses that sprite name for the WebSocket session. Before uploading, the client now checks the owner's latest `device_id + sprite_name` record on the server and skips the upload when the remote PNG/JSON/config hashes already match the local files. The public sprite detail endpoint remains available for listing and upload management, but room interaction no longer depends on looking up `/api/v1/sprites/{sprite_id}` during peer sync.

Join the random match pool or an invite room:

```bash
cd eggs
python3 scripts/egg_desktop.py remote
python3 scripts/egg_desktop.py remote room ABC123
python3 scripts/egg_desktop.py remote status
```

Leave or disable remote interaction:

```bash
cd eggs
python3 scripts/egg_desktop.py remote leave
python3 scripts/egg_desktop.py remote off
```

`remote` with no subcommand now uses the saved `remote.json` mode/room; when `mode=room` but the saved room code is empty, it falls back to random mode. `remote random` switches only the mode and preserves any saved room code for later reuse, while `remote status` shows the current remote config.

When remote interaction is enabled, the desktop companion still renders with the native Swift/Cocoa runtime on macOS. A separate Python sidecar process handles WebSocket sync and writes `~/.codex/eggs/remote-peers.json` (`connected`, `reconnecting`, `error`, and current peers) for the Swift runtime to consume.

When remote interaction is enabled and you run `python3 scripts/egg_desktop.py sprite <name>`, the client tries to ensure that sprite already exists on the server with matching hashes. If needed, it uploads first and only then sends the live sprite-switch message to the paired peer. If the remote check or upload fails, the local companion still changes sprite, but the remote session stays on the previous sprite so the peer never switches to a missing asset.

In `remote random` mode, the client enters a server-side waiting pool rather than a shared broadcast room. Once matched, the server places the two peers into a temporary private room used only for that pair.

## Remote Server

The standalone Go server is in `server/`:

```bash
cd server
go build -o eggs-server .
./eggs-server -addr :8787 -data ~/.codex/eggs-server -base-url http://localhost:8787
```

The server uses pure Go SQLite via `modernc.org/sqlite`, so the built binary does not require a target machine to have SQLite dynamic libraries installed.

## Sprite Tools

The extraction and merge tools live inside this skill at `tools/`.
When run with `--name <sprite>`, they write `<sprite>.png` and `<sprite>.json` to the output directory and automatically copy both files to `~/.codex/eggs/` for immediate playback. If extraction is run without `--name`, it writes `<input-name>_spritesheet.png/json`.

Build the extractor:

```bash
cd ..
mkdir -p .swift-module-cache
CLANG_MODULE_CACHE_PATH="$PWD/.swift-module-cache" \
swiftc -module-cache-path "$PWD/.swift-module-cache" \
  eggs/tools/extract_sprite.swift \
  -o /tmp/extract_sprite
```

Extract a bordered sheet:

```bash
/tmp/extract_sprite assets/6706ba10-3b4d-4a21-982c-4535dc931a3b.png \
  assets/sprites/state_a \
  --prefix state_a
```

Extract a sheet with a different cell height into a common frame canvas:

```bash
/tmp/extract_sprite assets/c1588c33-7053-4dfb-9f2a-fe572fb9d75f.png \
  assets/sprites/state_b \
  --prefix state_b \
  --frame-size 251
```

Merge extracted sheets:

```bash
CLANG_MODULE_CACHE_PATH="$PWD/.swift-module-cache" \
swiftc -module-cache-path "$PWD/.swift-module-cache" \
  eggs/tools/merge_spritesheets.swift \
  -o /tmp/merge_spritesheets

/tmp/merge_spritesheets assets/sprites/combined --name dino \
  assets/sprites/state_a/6706ba10-3b4d-4a21-982c-4535dc931a3b_spritesheet.json \
  assets/sprites/state_b/c1588c33-7053-4dfb-9f2a-fe572fb9d75f_spritesheet.json
```

## Sprite Sheet Rules

- PNG
- Frame size is defined by `<sprite>.json`; animation row names can be overridden in `~/.codex/eggs/config.json`.
- Regular grid
- The bundled sheet is 5 columns by 11 rows

## States

The bundled sheet is interpreted by row:

- Row 1: `unborn`
- Row 2: `ready`
- Row 3: `hatching`
- Row 4: `hatched`
- Row 5: `walk`
- Row 6: `sleep`
- Row 7: `eat`
- Row 8: `drink`
- Row 9: `play`
- Row 10: `roar`
- Row 11: `attack`
